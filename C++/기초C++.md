# 소수의 표현

float를 저장할 경우 소수점이 움직이는 부동(뜰 부)소수점을 사용하는데

부호 1자리, 지수 8자리, 유효숫자 23자리로 저장한다.

저장방법

1. 2진수 변환
2. 정규화 (1.xx X 2^n 으로 변환)
3. 자리수에 대입
   이때 지수는 부호없는 숫자로 판단하여 숫자 + 127로 변환함

   ex)-3.375 1. 3 + 0.375 = 0b11 + 0b0.011 = 0b11.011 2. 0b1.1011 \* 2^1 3. 1(부호) 1(지수) 1011(유효숫자) => 0b 1 10000000(128) 101'1000'0000'0000'0000'0000
   이는 조사식과 메모리를 통해 확인 가능 (Hex, 리틀 엔디안)

? 부동 소수점은 근사값임. (실수를 '=='으로 비교하는 것은 위험!)

# 문자열

영어와 숫가와 같은 경우는 아스키 코드로 할 수 있다.

하지만 한국어와 같이 여러 나라의 문자를 표기하기 위해서 유니코드를 이용한다.

- 가장 많은 번호를 차지하는 것이 한국어/중국어

- 유니코드는 utf8, utf16

  utf8

  - 알파벳, 숫자 1바이트 (ASCII 동일한 번호)

  - 유럽 지역의 문자는 2바이트

  - 한글, 한자 등은 3바이트

  utf16

  - 대부분의 문자가 2바이트로 한다.

  - 예외적인 문자는 4바이트로 사용

- Escape Sequence

  표기하기 애매한 문자들을 표기함 ex) tab, /등등

  - 대표적 문자

    | 표현되는 문자            | Escape Sequence | 아스키 코드 |
    | ------------------------ | --------------- | ----------- |
    | Tab                      | \t              | 9           |
    | Enter                    | \n              | 10          |
    | CarriageReturn (커서 <<) | \r              | 13          |
    | NULL                     | \0              | 0           |

문자열

문자들이 배열처럼 모여 있는 것,
정수 (1~8 바이트) 고정길이

끝은 null (0)

```Cpp
char str[] = {'h','e','l','l','o'};
```

- 해당 문자를 힙에 선언하면 문제 없으나 Stack에 선언시 문제가 된다.

  Stack에 넣으면, null문자를 넣어줘야 한다 (`\0`)

  ? 안정성을 위해 항상 null 붙여주는 것이 좋음

```Cpp
char str[] = "Hello"
```

위와 같이 null을 자동으로 해주는 ""도 생성 가능하다.
